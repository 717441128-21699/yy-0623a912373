# WordGuard · 码字断更助手

极简命令行工具，面向习惯用本地编辑器写作的资深网文作者和程序员作者。不打扰你的写作流程，只在需要时告诉你：**今天写了多少、还差多少、会不会断更、发之前要核对什么。**

## 特性

- 🪶 **零依赖纯 Python**：只用标准库，`pip install` 后即用
- ⌨️ **终端全流程**：初始化 → 进度 → 检查 → 发布核对 → 历史回看
- 📷 **基线快照机制**：`init` 时自动记录所有章节字数为基线，`progress` 默认只统计真正新增，`-b` 可显式把基线算入今日
- 📊 **草稿分类无关键词**：草稿目录下**所有章节默认按存稿/待发处理**，不再依赖文件名带 `draft`/`待发`；确认发布后自动从存稿中剔除
- ⏱️ **预计完成时间**：按当前速度反推几点能发，临近发布时间自动提醒
- 🚨 **断更四重检查**：今日发布线 / 存稿安全值 / 七天连续下滑 / 空目录或全发布警告
- ✅ **发布清单智能化**：
  - 悬念衔接双栏对比：上章末尾关键词+原句 / 本章开头命中上下文
  - 作者有话说 7 模式正则：同行冒号、下一行、括号、PS、多种别名
  - 本章钩子：38 词中文悬念词库 + 核心词双字滑窗匹配
- 📅 **写作日历回看**：30 天周视图热力图（·▁▃▆█ 五级）+ 达标率 + 最长连续更新 + JSON/CSV 导出

## 安装

```bash
# 在项目目录下执行（开发模式）
pip install -e .

# 或正式安装
pip install .
```

安装后有两个命令可用：`wordguard` 和简写 `wg`。

## 快速开始

### 1. 初始化

交互式填写（推荐）：

```bash
wg init
```

或者直接传参数：

```bash
wg init -n "我的小说名" -t 4000 -p 20:00 -d ~/Novel/drafts -s 3 -r 0.8
```

**初始化完成后会自动拍摄基线快照**：统计当前草稿目录下所有章节的字数，作为「已有的存量内容」。默认情况下存量**不计入**今日新增，避免把旧稿的几千字误算成今天写的。

参数说明：

| 参数 | 说明 | 默认 |
|------|------|------|
| `-n, --book-name` | 作品名称 | 交互输入 |
| `-t, --daily-target` | 日更目标字数 | 4000 |
| `-p, --publish-time` | 每日发布时间 `HH:MM` | 20:00 |
| `-d, --draft-dir` | 草稿目录路径 | 交互输入 |
| `-s, --safe-chapters` | 安全存稿章节数 | 3 |
| `-r, --publish-line-ratio` | 发布线比例（日更×比例） | 0.8 |

### 2. 查看 / 记录进度

```bash
# 扫描草稿目录，显示今日真正新增（基线之后的增量）、剩余目标、预计完成时间、章节动态
wg progress
# 或简写
wg p

# 想把初始化时已有的基线内容整体算进今日（比如今天把旧稿搬了过来）
wg progress --include-baseline
wg progress -b          # 简写，只生效一次，当天重复执行会自动跳过
```

手动记录一次字数（适合不想让工具扫描目录的场景）：

```bash
# 记录今天写了 1500 字
wg progress 1500

# 带章节名
wg progress 800 -c "第37章 夜探地牢"
```

输出包含：

- 进度条 + 完成百分比
- 今日新增（基线前后有明确区分）/ 剩余 / 是否到发布线
- 基线字数提示 + `-b` 用法
- 当前速度（字/小时）和预计完成时间
- 临近发布时间的「距发布还有 X 分钟」提醒
- 各章节字数动态（相对基线的增量展示 + 状态）
- 存稿概况（总章节、已发布数、存稿章节、可发章节、够写几天）

### 3. 断更检查

```bash
wg check
# 或简写
wg c
```

会逐条检查并分类到「正常项 ✓ / 警告项 ⚠」：

1. **今日发布线**：是否达到发布线字数，距发布时间是否紧迫
2. **存稿安全**：草稿章节数是否 >= `safe_chapters`，按**实际未发布章节数**计算
3. **七天趋势**：近 7 天是否连续 3 天以上字数下滑 + 高危断更检测
4. **发布时间**：距发布时间不足 3 小时但未到发布线 → 标紧迫
5. **昨日缺口**：昨日未达发布线 → 提示
6. **草稿目录**：空目录、全部已发布无新稿 → 单独告警

底部附 7 天柱状图（█ 达标 / ▓ 未达标），一眼看断更风险。

### 4. 发布前清单

自动挑选下一章待发布内容（草稿中最靠前的）：

```bash
wg publish
# 或简写
wg pub

# 手动指定章节路径或名称
wg publish ./drafts/chapter_42_draft.md
wg publish "第42章"
```

清单涵盖 5 项（按重要度分级）：

| 项 | 权重 | 说明 |
|----|------|------|
| 章节标题 | 必须 | 是否有有效的标题 |
| 字数达标 | 必须 | 是否达到发布线 + 差距/预计补稿时间 |
| 悬念衔接 | 必须 | 双栏对比：上章悬念关键词+原句 / 本章开头命中上下文片段 |
| 作者有话说 | 建议 | 同行冒号、下一行、括号、PS 多模式识别 + 字数统计 + 内容预览 |
| 本章钩子 | 建议 | 38 词悬念词库匹配 + 关键词和原句展示 |

清单通过后回车确认，**自动把该章标记为「已发布」**，后续 `progress` / `check` 的存稿统计不再包含这一章。

### 5. 写作日历 / 历史回看

```bash
# 查看最近 30 天写作日历
wg history
wg hist
wg h

# 指定天数
wg history -d 60

# 导出到草稿目录（JSON 含完整结构化数据，CSV 带 BOM 可直接用 Excel 打开）
wg history --export json
wg history --export csv
```

展示内容：

- **汇总**：总字数、过发布线天数、达标率、有写天数、有写日均、最长连续更新、当前连续 streak
- **周视图热力图**：5 级字符（·▁▃▆█）一眼看清每一天的产出
- **每日详情**：日期、星期、字数、达标标记、写作次数/章节数、基线标记 [基]

## 草稿目录约定

**不再依赖文件名关键词！** 草稿目录下所有 `.md` / `.txt` / `.markdown` 文件**默认全部算存稿/待发**。章节状态只在你执行 `wg publish` 确认发布后切换为「已发布」，后续自动从存稿中剔除。

推荐目录结构（按序号排序即可，文件名怎么命名都行）：

```
drafts/
├── 001_序章.md
├── 036_风云突变.md
├── 037_夜探地牢.md
├── 038_围追堵截.md
└── 039_破局.md
```

如果有已经发布过的旧稿，确认一遍 `wg publish 第一章` `wg publish 第二章`... 标记后就不会再被算进存稿了。发布后文件依然保留在目录里，只是状态被记录。

## 数据存储

- 配置文件：`~/.wordguard/config.json`
- 历史记录：`~/.wordguard/history.json`

`history.json` 包含：

| 键 | 说明 |
|----|------|
| `daily_records` | 每日汇总（总字数、达标标记、写作次数、章节数、是否含基线） |
| `chapter_snapshots` | 每次扫描的章节字数快照（用于日内增量检测） |
| `baseline_snapshots` | `init` 时刻的基线快照（跨日增量检测） |
| `published_chapters` | 已发布章节记录（发布日期、路径、字数） |
| `meta` | 元数据（基线创建时间、基线总字数等） |

一切本地化，不上传云端。需要迁移或备份直接复制整个 `~/.wordguard` 目录。

## 命令速查

```
wg                         # 命令总览
wg init                    # 初始化 + 基线快照
wg p                       # 查看今日进度（不含基线）
wg p -b                    # 把初始化基线整体计入今日（只生效一次）
wg p 1200                  # 手动记录 +1200 字
wg p 800 -c "第X章标题"    # 手动记录 +800 字并关联章节
wg check                   # 断更检查（含空目录/全发布告警）
wg pub                     # 发布前清单（悬念衔接双栏对比 + 作者有话说多模式）
wg pub ./xxx.md            # 指定章节清单
wg history -d 30           # 30 天写作日历 + 达标率 + 连续更新
wg history -e json         # 导出 JSON 备份到草稿目录
wg history -e csv          # 导出 CSV（带 BOM，Excel 直接打开）
```
