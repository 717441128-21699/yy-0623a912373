# WordGuard · 码字断更助手

极简命令行工具，面向习惯用本地编辑器写作的资深网文作者和程序员作者。不打扰你的写作流程，只在需要时告诉你：**今天写了多少、还差多少、会不会断更、发之前要核对什么。**

## 特性

- 🪶 **零依赖纯 Python**：只用标准库，`pip install` 后即用
- ⌨️ **终端全流程**：初始化 → 进度 → 检查 → 发布核对，不用开任何网站
- 📊 **智能字数扫描**：自动读取 `.md` / `.txt` / `.markdown` 文件，区分已发布与存稿
- ⏱️ **预计完成时间**：按当前速度反推几点能发，临近发布时间自动提醒
- 🚨 **断更三重检查**：今日发布线 / 存稿安全值 / 七天连续下滑趋势
- ✅ **发布前清单**：章节标题、悬念衔接、作者有话说、本章钩子

## 安装

```bash
# 在项目目录下执行（开发模式）
pip install -e .

# 或正式安装
pip install .
```

安装后有两个命令可用：`wordguard` 和简写 `wg`。

## 快速开始

### 1. 初始化

交互式填写（推荐）：

```bash
wg init
```

或者直接传参数：

```bash
wg init -n "我的小说名" -t 4000 -p 20:00 -d ~/Novel/drafts -s 3 -r 0.8
```

参数说明：

| 参数 | 说明 | 默认 |
|------|------|------|
| `-n, --book-name` | 作品名称 | 交互输入 |
| `-t, --daily-target` | 日更目标字数 | 4000 |
| `-p, --publish-time` | 每日发布时间 `HH:MM` | 20:00 |
| `-d, --draft-dir` | 草稿目录路径 | 交互输入 |
| `-s, --safe-chapters` | 安全存稿章节数 | 3 |
| `-r, --publish-line-ratio` | 发布线比例（日更×比例） | 0.8 |

### 2. 查看 / 记录进度

```bash
# 扫描草稿目录，显示今日新增、剩余目标、预计完成时间、章节动态
wg progress
# 或简写
wg p
```

手动记录一次字数（适合不想让工具扫描目录的场景）：

```bash
# 记录今天写了 1500 字
wg progress 1500

# 带章节名
wg progress 800 -c "第37章 夜探地牢"
```

输出包含：

- 进度条 + 完成百分比
- 今日新增 / 剩余 / 是否到发布线
- 当前速度（字/小时）和预计完成时间
- 临近发布时间的「距发布还有 X 分钟」提醒
- 各章节字数动态（区分「可发 / 撰写中 / 刚开」）
- 存稿概况（总章节、存稿章节、可发章节、够写几天）

### 3. 断更检查

```bash
wg check
# 或简写
wg c
```

会逐条检查并分类到「正常项 ✓ / 警告项 ⚠」：

1. **今日发布线**：是否达到发布线字数，距发布时间是否紧迫
2. **存稿安全**：草稿章节数是否 >= `safe_chapters`
3. **七天趋势**：近 7 天是否连续 3 天以上字数下滑
4. **发布时间**：距发布时间不足 3 小时但未到发布线 → 标紧迫
5. **昨日缺口**：昨日未达发布线 → 提示

底部附 7 天柱状图（█ 达标 / ▓ 未达标），一眼看断更风险。

### 4. 发布前清单

自动挑选下一章待发布内容（草稿中字数达到发布线的）：

```bash
wg publish
# 或简写
wg pub

# 手动指定章节路径或名称
wg publish ./drafts/chapter_42_draft.md
wg publish "第42章"
```

清单涵盖 5 项（按重要度分级）：

| 项 | 权重 | 说明 |
|----|------|------|
| 章节标题 | 必须 | 是否有有效的标题 |
| 字数达标 | 必须 | 是否达到发布线 |
| 悬念衔接 | 必须 | 上章悬念关键词是否在本章开头出现 |
| 作者有话说 | 建议 | 是否填写了「作者有话说」段落 |
| 本章钩子 | 建议 | 本章末尾是否检测到悬念措辞 |

清单过了后回车确认，自动把该章字数记入今日发布记录。

## 草稿目录约定

WordGuard 靠**文件名关键词**区分已发布 / 存稿，无需额外配置。文件名（小写）里包含 `draft` / `草稿` / `未发布` / `存稿` / `todo` / `待发` → 归为「存稿」；否则归为「已发布」。

示例目录结构：

```
drafts/
├── 001_序章.md                 # 已发布
├── 036_风云突变.md              # 已发布
├── 037_夜探地牢_draft.md        # 存稿（草稿）
├── 038_围追堵截_草稿.md          # 存稿
└── 039_破局_todo.md             # 存稿
```

## 数据存储

- 配置文件：`~/.wordguard/config.json`
- 历史记录：`~/.wordguard/history.json`（每次会话、每天汇总、章节快照）

不上传任何云端，一切本地化。

## 命令速查

```
wg                         # 命令总览
wg init                    # 初始化
wg p                       # 查看进度
wg p 1200                  # 记录 +1200 字
wg p 800 -c "第X章标题"     # 记录 +800 字并关联章节
wg check                   # 断更检查
wg pub                     # 发布前清单
wg pub ./xxx.md            # 指定章节清单
```
